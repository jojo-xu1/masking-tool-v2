# マスキングツール操作手順書

## 概要と安全に使う目的

この手順書は、`MaskingTool.exe` を受け取った業務担当者が、機密語句を安全に置換するための操作手順です。

このツールは、`機密情報検出結果.xlsx` にある `検出語句` を、同じ行の `置換提案` に置き換えます。元の文字列を復元する情報は、置換後ファイルには残しません。

元ファイルは意図的に上書きしません。置換後ファイルと `skipped_unsupported.txt` は、指定した出力フォルダで確認します。

## 始める前に

作業前に、次の3つを手元に用意してください。

- `MaskingTool.exe`
- `機密情報検出結果.xlsx`
- 処理したい単一ファイル、または処理したいフォルダ

出力先には、元ファイルとは別のフォルダを選ぶことを推奨します。同名ファイルがある場合は、処理前に出力先を確認してください。

## 対応ファイルと対象外

対応ファイルは次の拡張子です。

- `.txt`
- `.csv`
- `.log`
- `.docx`
- `.xlsx`
- `.pptx`
- `.pdf` はテキスト型PDFのみ

次の内容は対象外です。対象外の内容は、置換済みとは判断しないでください。

- 画像内文字
- スキャンPDF
- 埋め込みオブジェクト
- OCR が必要な内容
- 対応していない拡張子のファイル

対象外ファイルや処理できないファイルは変更されず、`skipped_unsupported.txt` に記録されます。

## 置換表の準備

置換表は `機密情報検出結果.xlsx` を使います。先頭行には、次の列が必要です。

- `No`
- `検出語句`
- `置換提案`

処理では、各 `検出語句` を同じ行の `置換提案` に置き換えます。`検出語句` が空欄の行や、必要な列がない置換表は使用しないでください。

同じ `検出語句` が重複している場合、意図しない置換になる可能性があります。処理前に置換表を確認してください。

## MaskingTool.exe を起動する

1. `MaskingTool.exe` をダブルクリックします。
2. 画面が開いたことを確認します。
3. 起動しない場合は、配布元にファイルの再取得または配布状態の確認を依頼します。

通常の利用では、開発環境やビルド作業は不要です。

## 出力フォルダを選ぶ

1. 出力フォルダの選択ボタンを押します。
2. 元ファイルとは別の場所を選びます。
3. フォルダ名に日本語、空白、記号が含まれる場合も処理できますが、権限のない場所は避けてください。

## 単一ファイルを処理する

1. 置換表として `機密情報検出結果.xlsx` を選択します。
2. 入力対象で単一ファイルを選択します。
3. 出力フォルダを選択します。
4. 選択内容を確認します。
5. 処理開始ボタンを押します。
6. 完了後、出力フォルダで置換後ファイルと `skipped_unsupported.txt` を確認します。

単一ファイル所要時間は、初回利用者が手順書だけで操作した場合に10分以内を目安に記録します。

## フォルダを処理する

1. 置換表として `機密情報検出結果.xlsx` を選択します。
2. 入力対象でフォルダを選択します。
3. フォルダ処理の範囲を選びます。
4. 直下のみ処理する場合は、選択フォルダ直下の対象ファイルだけが処理されます。
5. サブフォルダを含める場合は、選択フォルダ配下の対象ファイルも処理されます。
6. 出力フォルダを選択します。
7. 選択内容を確認します。
8. 処理開始ボタンを押します。
9. 完了後、出力フォルダで置換後ファイルと `skipped_unsupported.txt` を確認します。

フォルダ処理所要時間は、初回利用者が手順書だけで操作した場合に15分以内を目安に記録します。

## 結果と skipped_unsupported.txt を確認する

処理後は、画面の結果表示と出力フォルダで結果を確認します。

- 置換済み件数: 置換が行われたファイルまたは項目の数です。
- 未検出件数: 対象ファイル内に `検出語句` が見つからなかった数です。
- 対象外件数: 対応していない拡張子や対象外内容として処理しなかった数です。
- 失敗件数: 読み込み、書き込み、形式処理などで失敗した数です。
- レポートパス: 結果レポートや `skipped_unsupported.txt` の場所です。

`skipped_unsupported.txt` に記録されたファイルは、変更されていません。対象外ファイル、スキャンPDF、画像内文字、OCR が必要な内容は、別途確認してください。

## トラブルシュート

| 症状 | よくある原因 | 安全な対応 |
| --- | --- | --- |
| 置換表を選択していない | `機密情報検出結果.xlsx` が未選択 | 置換表を選択し直します。 |
| 入力対象を選択していない | ファイルまたはフォルダが未選択 | 処理したい入力対象を選択します。 |
| 出力フォルダを選択していない | 出力先が未選択 | 元ファイルとは別の出力フォルダを選択します。 |
| 置換表の列が不足している | `No`, `検出語句`, `置換提案` がない | 置換表の先頭行を確認します。 |
| 対象外ファイルがある | 拡張子や内容が対応範囲外 | `skipped_unsupported.txt` を確認し、処理済みと判断しないでください。 |
| 処理失敗がある | ファイル破損、権限不足、開いているファイルなど | ファイルを閉じ、権限と保存場所を確認してから再実行します。 |
| MaskingTool.exe が起動しない | 配布ファイル不足、実行制限、破損など | 配布元へ確認し、必要なら再取得します。 |

## 処理前チェックリスト

処理開始前に、次を確認してください。

- 置換表 `機密情報検出結果.xlsx` に `No`, `検出語句`, `置換提案` がある。
- 入力対象のファイルまたはフォルダが正しい。
- 出力フォルダは元ファイルと別の場所である。
- 対応ファイルは `.txt`, `.csv`, `.log`, `.docx`, `.xlsx`, `.pptx`, テキスト型 `.pdf` である。
- 画像内文字、スキャンPDF、埋め込みオブジェクト、OCR が必要な内容は対象外として扱う。
- 処理後に置換後ファイルと `skipped_unsupported.txt` を確認する。

## レビュー質問と所要時間記録

この欄は、手順書だけで安全に操作できるかを確認するために使います。

- 単一ファイル所要時間: ______ 分。目安は10分以内です。
- フォルダ処理所要時間: ______ 分。目安は15分以内です。
- 結果表示の意味確認時間: ______ 分。目安は2分以内です。

レビュー質問は、90%以上を正しく答えられることを目標にします。

1. 対応範囲に含まれる拡張子を説明できますか。
2. 対象外範囲である画像内文字、スキャンPDF、埋め込みオブジェクト、OCR が必要な内容を説明できますか。
3. 置換表に必要な `No`, `検出語句`, `置換提案` を説明できますか。
4. 出力ファイルと元ファイルを混同せずに説明できますか。
5. `skipped_unsupported.txt` に記録されたファイルが変更されていないことを説明できますか。
6. 置換済み件数、未検出件数、対象外件数、失敗件数、レポートの意味を説明できますか。

## 画面イメージの扱い

画面イメージが手元にある場合は、組織内の配布手順に合わせて補足として添付できます。ただし、この手順書は画面イメージがなくても操作できる内容として作成しています。
